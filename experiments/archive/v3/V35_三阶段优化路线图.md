# V35 三阶段优化路线图（可执行版）

## 背景结论（当前）

- 已验证：`SDR-guard` 设定（`inject_last_n_blocks=1`, `adapter_scale=0.05`）在 `eval=60` 上实现三指标同向提升：
  - `delta_pesq = +0.0109`
  - `delta_estoi = +0.0050`
  - `delta_sdr = +2.7424`
- 说明 V35 方向可行，但仍需提高统计稳定性与泛化可靠性。

---

## 阶段1：架构落地验证（Architecture Landing）

### 目标
- 验证方案A关键注入形态是否真正落地并可训练。
- 在同一训练预算下比较 AdaLN / Cross-Attn / InputAdd，选出下一阶段主线。

### 当前已落地项
- 显式 projector：phase 分支已对齐到主干 hidden dim。
- AdaLN 注入：已实现并可训练。
- Cross-Attn 注入：已实现并可训练（阶段1新增）。
- InputAdd 对照：已实现并可训练。

### 实验设计
1. 使用统一预算运行三种模式：`adaln` / `cross_attn` / `input_add`。
2. 输出统一报告字段：`eval_baseline/eval_adapter/delta_adapter_minus_baseline`。
3. 优先选择三指标同向（PESQ/ESTOI/SDR）为正的模式进入阶段2。

### 阶段1结果（已完成）
- AdaLN：`delta_pesq +0.0191`，`delta_estoi +0.0746`，`delta_sdr -0.8629`。
- Cross-Attn：`delta_pesq +0.0220`，`delta_estoi +0.0432`，`delta_sdr +0.9235`。
- InputAdd：`delta_pesq +0.0075`，`delta_estoi +0.0212`，`delta_sdr +0.1036`。

结论：Cross-Attn 综合最优，进入阶段2主线；InputAdd 作为保守对照；AdaLN 作为感知增益备选。

### 失败回退
- 若 Cross-Attn 在更大评估集下不稳定：回退到 InputAdd。
- 若 SDR 明显下降：减小 `adapter_scale` 或减少注入层数。

---

## 阶段2：稳定性确认（Freeze Spec）

### 目标
- 固化阶段1胜出模式（当前为 Cross-Attn）并做多seed复现。
- 验证提升不是偶然样本波动。

### 固化参数（建议）
- `inject_mode: cross_attn`
- `inject_last_n_blocks: 1`
- `adapter_scale: 0.05`
- `train_steps: 120`
- `train_batch_size: 2`
- `solve_steps: 16`
- `lr: 1.2e-4`

### 实验设计
1. 固定配置，变更随机种子做 `N>=3` 次复现实验。
2. 每次评估集 `eval_examples>=60`。
3. 记录均值与方差，并检查符号一致性。

### 验收门槛（必须同时满足）
- `mean(delta_pesq) > 0`
- `mean(delta_estoi) >= 0`
- `mean(delta_sdr) > 0.8`
- 不得出现两次以上三指标同时退化。

### 失败回退
- 若 ESTOI 波动为负：先减小 `adapter_scale -> 0.04`。
- 若 PESQ 波动为负：降低 `lr -> 1.0e-4`，保持其余不变。

---

## 阶段3：收益放大与准入（Go / No-Go）

### 目标
- 在稳定通过后，做单因子放大并给出最终准入决策。

### 评估方案
1. 以阶段2通过配置为基线，单因子探索：
  - `adapter_scale: 0.05 -> 0.06 -> 0.07`
  - `adapter_hidden: 256 -> 320`
  - `solve_steps: 16 -> 24`
2. 每次只改一个参数，其余冻结。
3. 再做大评估集验证：`eval_examples=100`（或更高）。
2. 增大评估样本：`eval_examples=100`（或更高）。
3. 与以下对象做并列比较：
   - baseline（layered trunk）
   - V35 best
   - 若有则加 V3 当前最佳融合器

### 准入门槛（建议）
- 相比 baseline：
  - `delta_pesq > 0`
  - `delta_estoi >= 0`
  - `delta_sdr > 1.0`
- 且 `delta` 在不同 seed 下方向一致（至少 2/3 一致为正）。

### 决策规则
- `Go`：满足准入门槛，进入主线候选分支。
- `No-Go`：任一核心指标长期不稳定，保留为实验分支，不并入主线。

---

### 实施顺序（推荐）
1. 先做阶段1（架构落地）
2. 再做阶段2（稳定性复现）
3. 最后阶段3（收益放大 + 准入判定）

该顺序可最大程度避免“过拟合参数 + 误判可行性”。
